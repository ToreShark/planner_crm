# AI Planner — PrimeLegal

Персональный AI-планировщик для адвоката. Пишешь в Telegram что угодно — ИИ анализирует, структурирует, ведёт планы и ежедневник за тебя.

## Идея

Никаких кнопок, форм и ручного планирования. Пишешь как думаешь:

- **"Заключил договор с +77782771481 из Уральска, аванс 100 тыс"** → ИИ создаёт задачу (done), записывает оплату, обновляет план дня и недели
- **"Завтра суд по Киламбекову в 10:00"** → задача на завтра, critical, синхронизация с планом недели
- **"Не успел жалобу Беденовой"** → ИИ находит задачу, переносит на завтра
- **"Итоги месяца"** → ИИ собирает статистику: выполнено, оплаты, что улучшить

ИИ (Claude) анализирует каждое сообщение:
- Прошедшее время ("заключил", "отправил") → автоматически **done**
- Будущее ("нужно составить", "завтра позвонить") → **pending** на нужную дату
- Упоминание денег → запись в оплаты
- Совпадение с существующей задачей → обновление статуса

## Архитектура

```
Telegram (текст) → Claude AI анализ → PostgreSQL → Планы дня/недели/месяца
                                          ↕
                                   Автосинхронизация
                                   день ↔ неделя
```

```
NestJS Backend
├── planner/                    # Ядро планировщика
│   ├── claude-planner.service  # Claude API: генерация планов, анализ текста
│   ├── context-builder.service # Сбор контекста: CRM, календарь, статистика
│   ├── plan-store.service      # PostgreSQL: CRUD планов, задач, оплат
│   ├── planner.controller      # REST API
│   ├── entities.ts             # TypeORM сущности
│   └── types.ts                # Интерфейсы
├── telegram/                   # Telegram-бот
│   ├── telegram-bot.service    # Бот: команды, кнопки, обработка текста
│   └── planner-cron.service    # Cron: утренний план, вечерний обзор, перенос задач
```

## Telegram-бот: команды

### Планирование (пошаговый wizard)
| Команда | Что делает |
|---------|-----------|
| `/plan` | Создать план на сегодня (energy → место → заметки → Claude генерирует) |
| `/week` | Создать план на неделю |
| `/month` | Создать план на месяц |

### Просмотр
| Команда | Что делает |
|---------|-----------|
| `/today` | План на сегодня (из памяти → БД → контрольная точка недели) |
| `/tomorrow` | План на завтра |
| `/thisweek` | План текущей недели с задачами по дням |
| `/history` | Планы за последние 7 дней с прогрессом |
| `/stats` | Статистика выполнения и оплаты |
| `/upcoming` | Все будущие задачи |
| `/tasks YYYY-MM-DD` | Задачи на конкретную дату |

### Итоги
| Команда | Что делает |
|---------|-----------|
| `/dayresults` | Итоги дня: выполнено, перенесено, оплаты |
| `/weekresults` | Итоги недели |
| `/weekreview` | AI-анализ итогов недели (пишешь текст → ИИ обновляет) |
| `/monthresults` | Итоги месяца |

### Свободный текст
Любое сообщение без команды → Claude анализирует и решает:
- Создать задачу (pending/done)
- Записать оплату
- Обновить существующую задачу

## Автоматизация (Cron)

| Время (Алматы) | Что происходит |
|----------------|---------------|
| **00:05** | Перенос незакрытых задач на сегодня + уведомление |
| **07:00** Пн-Сб | Claude генерирует утренний план дня |
| **21:00** Пн-Сб | Вечерний обзор: что сделано, что перенести |
| **10:00** Вс | Итоги недели |

## Синхронизация день ↔ неделя

Изменения в дневных планах автоматически обновляют checkpoints в плане недели:
- Нажал ✅ на задаче → checkpoint обновлён
- Добавил задачу текстом → появилась в плане недели
- Отменил задачу → исчезла из недельного обзора
- Сгенерировал план дня → неделя синхронизирована

## Правила планирования (System Prompt)

- **CRITICAL** — судебные дедлайны, заседания (пропуск = вред клиенту)
- **HIGH** — фокус недели/месяца
- **MEDIUM** — текущая работа, консультации
- **LOW** — контент, IT без дедлайна
- Cancelled задачи **удалены навсегда** — не кочуют, не воскресают
- Done задачи **не дублируются**
- Незакрытые задачи **автоматически переносятся** на следующий день
- Replan не трогает done/cancelled — только активные задачи

## REST API

### Генерация
- `POST /planner/day` — план дня через Claude
- `POST /planner/week` — план недели
- `POST /planner/month` — план месяца
- `POST /planner/replan` — перепланирование дня
- `POST /planner/review` — итоги дня

### Просмотр
- `GET /planner/day/:date` — план дня (`today`, `tomorrow`, `YYYY-MM-DD`)
- `GET /planner/week` — план текущей недели
- `PATCH /planner/task/:id/status` — обновить статус задачи
